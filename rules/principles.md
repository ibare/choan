# Principles

모든 코드에 항상 적용되는 핵심 원칙. 예외 없음.

## 1. 단일 책임
- 하나의 함수/컴포넌트/모듈은 하나의 역할만 수행한다
- 컴포넌트: 렌더링 로직과 비즈니스 로직을 분리한다
- 함수: 50줄을 초과하면 분리를 검토한다

## 2. 의존성 방향
- 계층: `engine` → `store` → `interaction`/`rendering` → `canvas`/`panels`/`pages`
- 하위 계층이 상위 계층을 import하지 않는다
- `engine/`은 React, Zustand 등 UI 레이어에 의존하지 않는다

## 3. 공유 인스턴스 재사용
- WebGL 렌더러, 좌표 시스템 등 공유 자원은 한 번 생성하여 재사용한다
- 동일한 인스턴스를 여러 곳에서 직접 생성하지 않는다

## 4. 중앙 집중 관리
- 설정값, 상수, 매직 넘버는 `constants/` 또는 `config/`에서 관리한다
- 컴포넌트나 함수 내부에 하드코딩하지 않는다

## 5. 불변 상태 업데이트
- Zustand 스토어의 상태는 항상 새 객체/배열로 업데이트한다
- 기존 상태를 직접 변경(mutate)하지 않는다

## 6. 타입 안전
- 함수의 입출력에 명시적 타입을 정의한다
- `any` 타입 사용을 금지한다. 불가피한 경우 `unknown`을 사용한다
- 암묵적 타입 변환에 의존하지 않는다
